# 设备联动使用说明

## 1. 概述

本系统设备联动模块基于 **Home Assistant (HA)** 智能家居平台实现，支持传感器实时监测、设备远程控制、告警自动推送三大核心能力。通过 HA WebSocket 长连接与后端 RuoYi 微服务架构对接，实现从"感知层 → 平台层 → 应用层"的完整闭环。

---

## 2. 系统架构

```
├─────────────────┤     ├─────────────────┤     ├─────────────────┤
│ 毫米波雷达       │────▶│ RuoYi 微服务     │────▶│ UniApp 小程序    │
│ 红外传感器       │     │ - 设备管理 API   │     │ - 设备控制页面   │
│ 人体存在传感器   │     │ - 告警推送服务   │     │ - 实时告警弹窗   │
│ 浸水传感器       │     │ - Webhook 网关   │     │ - 房屋状态看板   │
│ 智能音箱         │◀────│ - 数据聚合引擎   │     │ - 关怀模式 TTS  │
└─────────────────┘     └─────────────────┘     └─────────────────┘
         │                                               │
         └────────────── WebSocket ──────────────────────┘
```

## 3. 支持的设备类型

| 设备类型 | 英文标识 | 可控功能 | 联动场景示例 |
|---------|---------|---------|------------|

## 4. 前端使用指南

### 4.1 设备列表与状态查看

进入 **"我的房屋"** 页面，点击 **"设备状态"** Tab，可查看所有已绑定设备：

- 绿色圆点 = 设备在线
- 灰色圆点 = 设备离线
- 点击设备卡片进入 **设备详情页**

### 4.2 设备远程控制

在 **设备控制页** (`pages/device/control.vue`) 可进行以下操作：

#### 4.2.1 电源开关
- 点击 `switch` 组件切换设备开关状态
- 设备离线时开关自动禁用

#### 4.2.2 亮度调节（灯光设备）
- 拖动滑块调节亮度（0% - 100%）
- 仅在设备开启且在线时可用

#### 4.2.3 温度设置（空调设备）
- 点击 `+` / `-` 按钮调整目标温度
- 支持模式切换：制冷 / 制热 / 除湿 / 送风

### 4.3 传感器数据查看

在 **"我的房屋"** 页面点击 **"传感器"** Tab，可查看：

| 传感器 | 监测内容 | 数据用途 |
|-------|---------|---------|
| 全屋毫米波雷达 | 人体存在与活动状态 | 智能联动、跌倒检测 |
| 红外传感器 | 人体红外信号 | 区域入侵检测 |
| 人体存在传感器 | 高精度持续存在感知 | 长时间无人告警 |
| 声学异响传感器 | 异常声响（摔倒、破碎） | 紧急事件识别 |

## 5. 告警联动机制

### 5.1 告警规则配置

告警规则在 `uni/config/env.js` 中配置：

```javascript
alertRules: [
  {
    entityId: 'sensor.lumi_cn_918860765_mcn02_status_p_2_1',
    alertState: '自检',
    title: '银龄助手',
    content: '老人跌倒告警，请尽快查看！'
  },
  {
    entityId: 'event.xiaomi_cn_blt_3_1otprfbc0cc03_oh83w_submersion_sensor_event_e_2_1104',
    alertState: null,  // 任意非零状态均触发
    title: '银龄助手',
    content: '浸水告警，请尽快查看！'
  }
]
```

| 字段 | 说明 |
|-----|------|
| `entityId` | HA 中的实体唯一标识 |
| `alertState` | 触发告警的特定状态，`null` 表示任意非零/非 off 状态均触发 |
| `title` | 推送通知标题 |
| `content` | 推送通知内容 |

### 5.2 告警推送流程

```
传感器状态变化
    │
    ▼
HA WebSocket 推送
    │
    ▼
前端 ha-fall-alert.js 监听
    │
    ▼
匹配告警规则 ──否──▶ 忽略
    │是
    ▼
发送本地推送 (APP) / 弹窗提示 (H5)
```

### 5.3 推送方式

| 运行环境 | 推送方式 | 效果 |
|---------|---------|------|
| APP (APP-PLUS) | `plus.push.createMessage` | 系统级通知栏推送，带声音 |
| H5 | `uni.showModal` | 页面模态弹窗 |

## 6. 后端 API 接口

### 6.1 设备管理接口

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/system/wyxdevice/list` | GET | 获取设备列表 |
| `/system/wyxdevice/{id}` | GET | 获取设备详情 |
| `/system/wyxdevice` | PUT | 控制设备（开关、亮度、温度等） |

### 6.2 控制设备请求示例

```javascript
import { controlDevice } from '@/api/device.js'

// 开灯并设置亮度为 80%
controlDevice(deviceId, {
  status: 1,      // 1=开启, 0=关闭
  value: 80       // 亮度值 0-100
})

// 关闭空调
controlDevice(deviceId, {
  status: 0
})
```

### 6.3 Webhook 告警推送

后端可通过 Webhook 将高危事件推送到 HA，触发音箱播报：

```bash
curl -X POST http://<HA_IP>:8123/api/webhook/harmony_emergency \
  -H 'Content-Type: application/json' \
  -d '{
    "room_id": "3-201",
    "event_type": "跌倒",
    "level": "一级高危"
  }'
```

## 7. Home Assistant 配置

### 7.1 接入前提

1. HA 服务已部署并运行（默认端口 `8123`）
2. 已获取 **长期访问令牌** (Long-Lived Access Token)
3. 前端 `env.js` 中已正确配置 `haHost`、`haPort`、`haAccessToken`

### 7.2 自动化配置示例

在 HA 的 `configuration.yaml` 中配置：

```yaml
alias: "[Harmony] 鸿蒙紧急告警处置"
description: "接收后端微服务推送的一级高危事件，联动小艺音箱播报"
trigger:
  - platform: webhook
    webhook_id: harmony_emergency
    allowed_methods:
      - POST

action:
  - service: media_player.volume_set
    data:
      volume_level: 0.8
    target:
      entity_id: media_player.xiaomi_wifispeaker_l05b

  - service: tts.xiaomi_say
    data:
      entity_id: media_player.xiaomi_wifispeaker_l05b
      message: >
        紧急告警。{{ trigger.json.room_id }}房间发生{{ trigger.json.event_type }}事件，
        风险等级{{ trigger.json.level }}，请立即前往查看。
```

### 7.3 配置检查清单

| 检查项 | 操作 |
|-------|------|
| HA YAML 已配置 | 重启 HA 或 **自动化 → 重新加载** |
| Webhook 可访问 | 执行 `curl` 测试请求 |
| 小艺音箱在线 | HA 中 `media_player.xiaomi_wifispeaker_l05b` 状态为 `idle` |
| Token 有效 | 确认 `haAccessToken` 未过期 |

## 8. 关怀模式联动

开启关怀模式后，系统会自动：

1. **文字放大**：页面文字缩放至 120%
2. **语音播报**：进入页面时自动朗读页面标题
3. **简化交互**：按钮增大，操作反馈增强

开启方式：在 **"我的" → 设置** 中打开 **"关怀模式"** 开关。

## 9. 常见问题

### Q1: 设备显示离线怎么办？
- 检查设备电源是否正常
- 确认设备与 HA 网关在同一局域网
- 在 HA 前端查看设备实体状态

### Q2: 告警推送收不到？
- 检查 `env.js` 中 `haAccessToken` 是否配置正确
- 确认 HA WebSocket 端口（默认 8123）未被防火墙拦截
- 查看浏览器/APP 通知权限是否开启

### Q3: 控制设备无响应？
- 确认设备在线状态
- 检查后端 API `/system/wyxdevice` 是否返回成功
- 查看 HA 日志确认指令是否到达

### Q4: 如何添加新的传感器告警？
在 `uni/config/env.js` 的 `alertRules` 数组中新增规则对象，填写正确的 `entityId` 和触发条件即可。
